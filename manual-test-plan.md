# خطة اختبار يدوي شاملة للمشروع

الملف: manual-test-plan.md
الهدف: إعداد خطة اختبار يدوية دقيقة تغطي الوظائف الرئيسية لتطبيق التجارة الإلكترونية بهدف تمكين تنفيذ الاختبارات، تسجيل النتائج، وإرسال عيوب قابلة للتنفيذ.

---

## 1. ملخص
خطة اختبار يدوي تغطي: التسجيل/تسجيل الدخول، إدارة الحساب، عرض المنتجات والبحث، العربة، الدفع وطلب الشراء، لوحة الإدارة، مهام الخلفية، التكاملات (إيميل، بوابة دفع)، والأمان.

## 2. نطاق الاختبار
- صفحات عامة: index.php، landing_page.php، faq.php
- التسوق: product-category.php، premium-product-page.php، store.php، search-result.php
- عربة/شراء: buy-now.php، save-incomplete-order.php، reorder_form*.php، checkout صفحات التأكيد
- المستخدم: registration.php، login.php، edit-profile.php، customer-order.php
- إدارة: admin/، super-admin/
- مهام وخلفيات: update-abandoned-cart.php، update-incomplete-order.php، incomplete-orders.php
- سكربتات تحديث/DB: update_db_*.php

## 3. بيئة الاختبار وإعدادها
- نظام: Windows + XAMPP (Apache + MySQL) أو نسخة مماثلة على خادم اختبار
- استيراد قاعدة بيانات اختبار من: `boomtsvp_ecommerceweb (1).sql` أو `database_structure.sql`
- تحديث إعدادات الاتصال بقاعدة البيانات إلى بيانات الاختبار (ملف التكوين في inc/ أو ملف config.php إن وُجد)
- تمكين إرسال البريد عبر SMTP اختبارية أو استخدام أداة مثل MailHog
- وضع بوابة الدفع في وضع الاختبار (test/sandbox)

## 4. بيانات الاختبار
- حسابات:
  - user@example.com / Password1!  (مستخدم عادي)
  - admin@example.com / AdminPass1! (مدير)
  - staff@example.com / StaffPass1!
- منتجات:
  - منتج متوفر، كمية كافية
  - منتج بنفاذ المخزون
  - منتج بسعر ترويجي
- عناوين شحن متعددة، بطاقات دفع اختبارية

## 5. أدوار الاختبار
- زائر (غير مسجل)
- مستخدم مسجل
- موظف دعم
- مدير النظام (Admin)

## 6. تصنيف الشدة (Severity)
- حرجة (Critical): فشل مسار الدفع/إنشاء طلب
- عالية (High): لا يمكن إضافة للعربة أو عرض صفحة المنتج
- متوسطة (Medium): أخطاء واجهة أو رسائل خاطئة
- منخفضة (Low): أخطاء تجميلية أو نصوص

## 7. منهجية الاختبار
- كل حالة تحتوي: معرف، الهدف، شروط سابقة، خطوات تنفيذية، نتيجة متوقعة، شدة
- نفّذ حالات إيجابية وسلبية وحدود المدخلات
- سجّل النتائج بصيغة CSV أو في تتبع العيوب (Jira/GitHub Issues)

## 8. حالات الاختبار الأساسية

### تسجيل وتسجيل الدخول
- AUTH-01: تسجيل مستخدم جديد
  - شرط: صفحة registration.php تعمل
  - خطوات: فتح الصفحة، ملء الاسم/البريد/كلمة السر، إرسال
  - متوقع: إعادة توجيه لتأكيد/صفحة تسجيل الدخول، استقبال رسالة تأكيد عبر الإيميل
  - شدة: عالية

- AUTH-02: تسجيل دخول ناجح
  - خطوات: الدخول ببيانات صحيحة
  - متوقع: الوصول إلى dashboard أو الصفحة الرئيسية مع ظهور اسم المستخدم
  - شدة: حرجة

- AUTH-03: تسجيل دخول خاطئ
  - خطوات: إدخال بريد صحيح وكلمة خاطئة
  - متوقع: رسالة خطأ واضحة، لا جلسة مُنشأة
  - شدة: متوسطة

- AUTH-04: استعادة كلمة المرور
  - خطوات: الدخول إلى forget-password.php وإرسال البريد
  - متوقع: استلام رابط إعادة تعيين وكود صالح للتفعيل
  - شدة: عالية

### الملف الشخصي
- PROFILE-01: تعديل الملف الشخصي
  - خطوات: تسجيل الدخول -> edit-profile.php -> تعديل الحقول -> حفظ
  - متوقع: بيانات محفوظة وتعرض في الصفحة
  - شدة: متوسطة

### تصفح المنتجات والبحث
- PROD-01: عرض تصنيف منتجات
  - خطوات: فتح product-category.php مع فئة محددة
  - متوقع: عرض قائمة المنتجات، صور، أسعار، زر إضافة للعربة
  - شدة: متوسطة

- PROD-02: صفحة تفاصيل منتج
  - خطوات: فتح صفحة المنتج (premium-product-page.php)
  - متوقع: وصف، سعر، مخزون، صور، خيارات (إن وجدت)
  - شدة: متوسطة

- PROD-03: البحث
  - خطوات: البحث بكلمة مفتاحية من search-result.php
  - متوقع: نتائج متطابقة أو رسالة "لم يتم العثور"
  - شدة: متوسطة

### العربة (Cart)
- CART-01: إضافة منتج للعربة
  - خطوات: من صفحة المنتج اضغط "أضف إلى العربة"
  - متوقع: عدد المنتجات في العربة يزيد، السعر الإجمالي محسوب
  - شدة: حرجة

- CART-02: تحديث الكمية
  - خطوات: تغيير كمية في صفحة العربة ثم تحديث
  - متوقع: إعادة حساب المجموع والضرائب والشحن
  - شدة: عالية

- CART-03: إزالة منتج
  - خطوات: إزالة منتج من العربة
  - متوقع: العنصر يختفي والمجموع يتحدث
  - شدة: عالية

- CART-04: حفظ طلب ناقص
  - خطوات: أثناء ملء الطلب استخدم save-incomplete-order.php
  - متوقع: يتم تخزين حالة الطلب وظهوره في incomplete-orders.php
  - شدة: متوسطة

### الدفع وإتمام الطلب
- CHECKOUT-01: تنفيذ طلب ناجح
  - خطوات: اتبع خطوات الخروج، أدخل معلومات الشحن، اختر طريقة الدفع التجريبية، أكمل الدفع
  - متوقع: إنشاء سجل طلب، توجيه إلى payment-success.php، استقبال بريد تأكيد
  - شدة: حرجة

- CHECKOUT-02: فشل الدفع
  - خطوات: محاكاة رفض البطاقة
  - متوقع: رسالة خطأ واضحة، لم يتم إنشاء طلب دائم أو يتم وسمه كـ "فشل"، استعادة المخزون إن حجز
  - شدة: حرجة

- CHECKOUT-03: "اشتر الآن" (Buy Now)
  - خطوات: استخدام buy-now.php من صفحة المنتج
  - متوقع: تخطي شاشة العربة والانتقال المباشر لصفحة الدفع مع المنتج فقط
  - شدة: عالية

### الطلبات
- ORDER-01: تأكيد الطلب
  - خطوات: عرض order-confirmation.php بعد الدفع
  - متوقع: تفاصيل الطلب، رقم الطلب، حالة الدفع
  - شدة: عالية

- ORDER-02: تاريخ طلبات المستخدم
  - خطوات: زيارة customer-order.php
  - متوقع: قائمة الطلبات مع تفاصيل وروابط لعرض كل طلب
  - شدة: متوسطة

### لوحة الإدارة
- ADMIN-01: دخول المشرف
  - خطوات: تسجيل دخول عبر لوحة admin
  - متوقع: الوصول لأدوات الإدارة
  - شدة: حرجة

- ADMIN-02: إضافة/تعديل منتج
  - خطوات: إنشاء أو تعديل منتج عبر patch_product_edit.php / لوحة إدارة
  - متوقع: المنتج يظهر في المتجر مع البيانات الجديدة
  - شدة: عالية

- ADMIN-03: مراجعة الطلبات الناقصة
  - خطوات: فتح incomplete-orders.php وراجع الحالات
  - متوقع: عرض الطلبات المحفوظة وإمكانية استعادتها أو حذفها
  - شدة: متوسطة

### مهام الخلفية وجدولة المهام
- BG-01: تنظيف العربة المهجورة
  - خطوات: تشغيل update-abandoned-cart.php يدوياً
  - متوقع: تحديد وإزالة العرب التي تجاوزت مهلة، إرسال تنبيهات إن لزم
  - شدة: متوسطة

- BG-02: تحديث الطلبات الناقصة
  - خطوات: تشغيل update-incomplete-order.php
  - متوقع: تغيير حالات الطلبات القديمة إلى منتهية أو إرسال تذكير
  - شدة: متوسطة

### التكامل والإيميل
- INT-01: رسائل التسجيل والطلب
  - خطوات: تسجيل مستخدم / إتمام طلب
  - متوقع: استقبال رسائل بصيغ صحيحة وروابط تعمل
  - شدة: عالية

- INT-02: بوابة الدفع (Webhooks)
  - خطوات: محاكاة Webhook من بوابة الدفع
  - متوقع: استقبال الويب هوك وتحديث حالة الطلب تلقائياً
  - شدة: حرجة

### الأمان
- SEC-01: اختبار SQL Injection (سلبية)
  - خطوات: إرسال مدخلات مشبوهة في حقول البحث/تسجيل الدخول
  - متوقع: تطبيق يتحقق ولا يظهر استثناءات/تسريبات
  - شدة: حرجة

- SEC-02: اختبار XSS
  - خطوات: إدخال سكربت في حقل اسم المنتج أو التعليقات
  - متوقع: يتم تعقيم المدخلات أو ترميزها، لا تنفيذ سكربت
  - شدة: حرجة

### توافق ومتصفح
- COMP-01: اختبار المتصفحات الرئيسية (Chrome, Firefox, Edge, Safari mobile)
  - خطوات: تنفيذ مسارات رئيسية على كل متصفح
  - متوقع: عرض صحيح ووظائف أساسية تعمل
  - شدة: متوسطة
